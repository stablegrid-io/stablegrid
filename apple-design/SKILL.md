# Apple Design

Create premium-feeling web interfaces that stay minimal, intentional, and structurally clean.

## Workflow

1. Define intent and focal point.
- State one screen goal and one primary CTA.
- Pick one dominant visual idea per viewport.

2. Simplify structure and content.
- Remove repeated sections, duplicate messages, and competing CTAs.
- Merge or delete components that perform the same job.
- Reduce copy to short, confident lines.

3. Build a restrained visual system.
- Set consistent typography scale with limited font weights.
- Apply a stable spacing rhythm with generous section padding.
- Use neutral foundations with one controlled accent color.
- Add cards only when grouping improves clarity.
- Keep one icon style and use icons only when they clarify.

4. Implement premium interaction quality.
- Keep motion purposeful and understated.
- Prefer soft fades/transforms; avoid flashy or bouncy effects.
- Ensure semantic HTML, keyboard access, and clear interaction states.
- Keep mobile and desktop layouts equally intentional.

5. Polish and cut.
- Run the redundancy checklist from the reference file.
- Remove anything that weakens hierarchy, clarity, or cohesion.
- Keep a short note of what was intentionally removed.

## Output Contract

When using this skill, return:
1. Brief design direction summary.
2. Implementation.
3. Short note listing intentional simplifications/removals.
