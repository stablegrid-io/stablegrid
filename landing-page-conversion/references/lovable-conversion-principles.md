# Lovable Conversion Principles

Use this reference to keep recommendations concrete and measurable.

## Fast Audit Rubric

Score each area `0` (poor) to `2` (strong):

- Offer clarity in 5 seconds
- Headline specificity and outcome focus
- Primary CTA clarity and visibility
- Form friction (field count + effort)
- Social proof specificity and relevance
- Mobile usability and readability
- Load speed and perceived responsiveness

Treat any score below `1` as a priority fix.

## Messaging Rules

- State who the product is for, what result it delivers, and how quickly/value level.
- Replace generic superlatives with specific outcomes.
- Pair each major claim with evidence.
- Keep hero copy scannable: one clear headline, one support line, one primary CTA.

## CTA Rules

- Use one primary CTA per page intent.
- Repeat the same primary CTA label in major sections.
- Match CTA to funnel stage:
- Low commitment: `Try free`, `Get the guide`
- Mid commitment: `Book a demo`, `See pricing`
- High commitment: `Start now`, `Buy now`

## Form Friction Rules

- Keep forms to essential fields only.
- Remove optional fields from first step when possible.
- Prefer multi-step only when it reduces perceived effort.
- Use context-appropriate inputs on mobile (email keyboard, numeric keyboard).

## Social Proof Rules

Prioritize this order:

1. Quantified outcome testimonials with role/company
2. Recognizable customer logos with context
3. Case-study snippets tied to the same audience segment
4. Ratings/reviews with enough sample size

Avoid anonymous or non-specific praise.

## Mobile Rules

- Design and validate on mobile first.
- Keep line length readable and avoid dense paragraphs.
- Keep primary CTA reachable without precision taps.
- Reduce visual clutter and non-essential motion.
- Ensure forms are keyboard-friendly and quick to complete.

## Performance Rules

- Compress and right-size images.
- Defer or remove non-critical scripts.
- Minimize render-blocking assets.
- Limit heavy hero media unless it materially improves conversion.

Prioritize perceived speed on first view.

## Experiment Sequencing

Run tests in this order unless evidence suggests otherwise:

1. Headline/value proposition clarity
2. Form simplification
3. CTA wording and placement
4. Social proof placement/content
5. Page structure/layout refinements

Run one major variable at a time.

## Experiment Spec Template

Use this template for each experiment:

- Hypothesis: If we change `X`, then conversion rate will increase because `Y`.
- Variant: Exact copy/layout change.
- Primary metric: Main conversion event.
- Secondary metrics: Bounce rate, scroll depth, form start/complete.
- Runtime rule: Run full business cycles and stop only after minimum sample threshold.

## Deliverable Format

When producing recommendations, include:

1. `Priority Fixes (Now)` with impact/effort
2. `Section Rewrite` (hero, proof, CTA, form, FAQ/risk reversal)
3. `UI/UX Changes` with mobile specifics
4. `Measurement Plan` (events + dashboard fields)
5. `Experiment Backlog` (next 3-5 tests)
